# 终端触摸滚动优化设计

**日期**: 2026-03-22

## 问题描述

移动端浏览器访问终端时，用户在终端区域内用手指上下滚动查看内容，触摸动作与浏览器默认的页面滚动行为冲突，导致滚动体验时好时坏、不可靠。

## 需求

- 终端区域内：禁用浏览器页面滚动，所有触摸滑动用于终端内容滚动
- 终端区域外：保持浏览器原有触摸行为不变
- 保留底部"滚动到底部"按钮

## 解决方案

采用 CSS `touch-action` 属性控制触摸行为。

### 实现

修改文件：`packages/web/src/components/TerminalTab.vue`

在 `.terminal-wrapper` 样式中添加：

```css
.terminal-wrapper {
  position: absolute;
  inset: 0;
  touch-action: pan-y;
}
```

### 原理

`touch-action: pan-y` 告诉浏览器：
- 允许用户在元素上垂直方向滑动
- 禁用浏览器的页面刷新手势、前进/后退手势
- 浏览器不再拦截触摸事件，xterm.js 可正常处理滚动

### 影响范围

- 仅影响终端区域（`.terminal-wrapper`）
- 页面其他区域（顶部工具栏、底部快捷键栏、设置页面等）不受影响

## 验收标准

1. 在终端区域内上下滑动，可以可靠地滚动终端内容
2. 不再出现浏览器页面滚动与终端滚动冲突的情况
3. 底部"滚动到底部"按钮功能正常
4. 页面其他区域的触摸行为不受影响
